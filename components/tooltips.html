Tooltips

Tooltips

Display container for Tooltip content. Has a directional arrow dependent on its placement.

💡

Check out our ladle for more info on this component.

Import

import {Tooltip, TooltipTrigger} from '@comfortdelgro/react-compass'

or

import {Tooltip, TooltipTrigger} from '@comfortdelgro/react-compass/tooltip'

We recommend the second method to import components for the benefits of tree shaking (opens in a new tab).

Content

The TooltipTrigger accepts exactly two children: the element which triggers the display of the Tooltip and the Tooltip itself. The trigger element must be the first child passed into the TooltipTrigger. All content should be internationalized.

Basic tooltip

Hover me
<TooltipTrigger>
  <a href='#'>Hover me</a>
  <Tooltip>Tooltip</Tooltip>
</TooltipTrigger>

Title

Render title of the tooltip

Hover me
<TooltipTrigger>
  <a href='#'>Hover me</a>
  <Tooltip title='Title' dismissible>
    Tooltip
  </Tooltip>
</TooltipTrigger>

Tooltip Delay

Adjust delay of tooltip after hovering the trigger

Hover me
<TooltipTrigger delay={1000}>
  <a href='#'>Hover me</a>
  <Tooltip>Tooltip</Tooltip>
</TooltipTrigger>

Dismissible

Enable a close button in the tooltip

Hover me
<TooltipTrigger>
  <a href='#'>Hover me</a>
  <Tooltip dismissible>Tooltip</Tooltip>
</TooltipTrigger>

Customization

Child element

The tooltip needs to apply DOM event listeners to its child element. If the child is a custom React element, you need to make sure that it spreads its props to the underlying DOM element.

const MyComponent = React.forwardRef(function MyComponent(props, ref) {
  //  Spread the props to the underlying DOM element.
  return (
    <div {...props} ref={ref}>
      Custom Component
    </div>
  )
})
 
// ...
<TooltipTrigger>
  <MyComponent/>
  <Tooltip dismissible>Tooltip</Tooltip>
</TooltipTrigger>

Styling

You can style React Compass's components with the css prop.
With css prop, you can basically do anything that css can do.

Hover me
<TooltipTrigger>
  <a href='#'>Hover me</a>
  <Tooltip
    title={<p style={{fontSize: '20px', margin: 0}}>Title</p>}
    dismissible
    css={{backgroundColor: 'black'}}
  >
    Lorem ipsum dolor sit amet consectetur adipisicing elit. Recusandae
    blanditiis laborum commodi mollitia officiis minus ipsum totam fuga odit,
    dicta magnam numquam reprehenderit et sequi dignissimos iusto deleniti.
    Quia, modi
  </Tooltip>
</TooltipTrigger>

Max-width

The Tooltip wraps long text by default to make it readable. You can customize it to fit your desire

Hover me
<TooltipTrigger>
  <a href='#'>Hover me</a>
  <Tooltip title={'Title'} css={{maxWidth: 'none'}}>
    Lorem ipsum dolor sit amet consectetur adipisicing elit. Recusandae
    blanditiis laborum commodi mollitia officiis minus ipsum totam fuga odit,
    dicta magnam numquam reprehenderit et sequi dignissimos iusto deleniti.
    Quia, modi
  </Tooltip>
</TooltipTrigger>

Placement

The Tooltip's placement with respect to its trigger element can be adjusted using the placement prop. See the props table below for a full list of available placement combinations

Hover me
<TooltipTrigger placement='right'>
  <a href='#'>Hover me</a>
  <Tooltip>Open on the right</Tooltip>
</TooltipTrigger>

Offset

Hover me
<TooltipTrigger offset={50}>
  <a href='#'>Hover me</a>
  <Tooltip>This will shift down.</Tooltip>
</TooltipTrigger>
Hover me
<TooltipTrigger offset={{crossAxis: 50}}>
  <a href='#'>Hover me</a>
  <Tooltip>This will shift over to the right.</Tooltip>
</TooltipTrigger>

Props

TooltipTrigger Props:

NameTypeDefaultDescription
children*[ReactElement, ReactElement]—The first child must be a trigger, the second child must be Tooltip component
offsetnumber | {mainAxis: number, crossAxis: number}The additional offset applied along the main axis and(or) cross axis between the element and its anchor element.
delaynumber0The delay time for the tooltip to show up
trigger'focus' | nullnullBy default, opens for both focus and hover. Can be made to open only for focus.
isOpenboolean—Whether the overlay is open by default (controlled)
defaultOpenboolean—Whether the overlay is open by default (uncontrolled).
placementPlacementbottomThe placement of the element with respect to its anchor element.
shouldFlipbooleantrueWhether the element should flip its orientation (e.g. top to bottom or left to right) when there is insufficient room for it to render completely.
withArrowbooleantrueWhether the element should have an arrow pointed to anchor element
disableInteractivebooleanfalseTooltips are interactive by default (to pass WCAG 2.1 success criterion 1.4.13). It won't close when the user hovers over the tooltip. You can disable this behavior (thus failing the success criterion which is required to reach level AA)
onOpenChange(isOpen: boolean) => void—Handler that is called when the overlay's open state changes.
cssCSS—The system prop that allows defining system overrides as well as additional CSS styles.
